Getting started

You need at least PHP version 8.1+ installed for this Script to work properly. It is strongly recommended that you always use the latest stable version of PHP to ensure all known bugs and security issues are fixed. Other getting_started:

  • PHP version 8.1 or greater. But for maximum speed gain we recommend you to use PHP version 7.3+.
  • MySQL version 5.6 or greater OR MariaDB version 10.0 or greater.
  • PHP memory limit of 128 MB or greater.

Recommended PHP configuration limits

  • max_execution_time 900
  • memory_limit 512M
  • post_max_size 256M
  • upload_max_filesize 500M
  • max_input_time = 900
  • max_input_vars = 1000

Server getting_started: https://make.wordpress.org/hosting/handbook/handbook/server-environment

500 ERROR Before Installing

  1. *env file is existed - Use the script that is installed already. You should download the Newest version for New install
  2. *env file is not existed - You should check the permission of the folder
  3. PHP version is less than 8.x - GoTrip is working fine in version 8.x.x
  4. *env file is existed - Use the script that is installed already. You should download the Newest version for New install
  5. Files missing while move from GoTrip folder to public_html

1. Show the Hidden files

2. Find the Hidden file and upload it again

500 Error after installing

1. The reason of 500|Server error

  1. TURN ON the Debug to know the reason of issues
  2. Go to public_html/.evn file, open to turn TRUE the debug
  1. 1 - Access Denied for User: After installing the Deny Warrning is showed because of the connection
  2. 2 - The database is not existed
  3. 3 - The database is existed but the user is not existed
  4. 4 - The database is existed but the password is not correct
  5. 5 - After installing, the 500|Server Error is showed, try to TURN ON the debug to see the reason of issues

Fix 500 ERROR

1 - Delete the .env file + storage/installed.php file

2 - Create new DB and new User

- Create new DB

- Create new User

-Assign Database to User
